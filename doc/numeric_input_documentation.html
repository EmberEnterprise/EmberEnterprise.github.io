    Numeric input - Documentation

    Easy to use and customizable numeric input component.

    API

    Attribute Type Value Description
    disabled boolean false default: Can interact with component
    true Input and buttons are unusable and un-clickable
    onChange string (action <action-name>) Triggers associated action when the input value is changed
    value number <numeric-value> Value to bind to the input field
    max number null default: Input has no maximum value limit
    <numeric-value> Maximum value accepted
    min number null default: Input has no minimum value limit
    <numeric-value> Minimum value accepted
    format string null default: Simple numbers are displayed
    percentage The '%' sign is displayed next to the value inside the input field
    currency The '$' sign is displayed in front of the value inside the input field
    custom The string provided in the 'customFormatUnit' field is displayed next to the value inside the input field
    currencyUnit string $ default: Ignored when 'format' type is not 'currency'
    <currency-unit> The given string is displayed in front of the value inside the input field. Examples of currency units are '€', '£'. Ignored when 'format' type is not 'currency'
    customFormatUnit string null default: Ignored when 'format' type is not 'custom'
    <custom-format-unit> The given string is displayed next to the value inside the input field. Examples of custom unit types are 'kg', 'cm'. Ignored when 'format' type is not 'custom'
    decimals number 0 default: No decimals are displayed
    <numeric-value> The give number of decimals are displayed

    Examples

    Simple

    {{e-numeric-input value=20 min=0 max=100}}

    Currency format

    {{e-numeric-input value=20 min=0 max=100 format='currency'}}

    Custom format

    {{e-numeric-input value=20 min=0 max=100 format='custom' customUnitType='kg'}}

    Percentage format with 2 decimals

    {{e-numeric-input value=20 min=0 max=100 format='percentage' decimals=2}}
